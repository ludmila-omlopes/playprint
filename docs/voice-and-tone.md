# Voice And Tone

filazo treats a game library as abundance, not an assignment. The product voice is calm, specific, and forgiving: gentle over gamified.

## Principles

- Prefer shelf, library, curiosity, and rest language over task, debt, or pressure language.
- Treat large counts as breadth and abundance, never as a problem to solve.
- Offer next steps as invitations. Avoid commands that imply the user has fallen short.
- Separate story completion from achievement mastery. Credits rolled is enough.
- Keep copy concrete and useful when something needs action, especially sync and import errors.

## Status Vocabulary

| Internal status | Display label | Tone note |
| --- | --- | --- |
| `BACKLOG` | on the shelf | Present without pressure. |
| `OWNED` | owned | Plain and factual. |
| `WISHLIST` | still curious | Curiosity, not obligation. |
| `PLAYING` | playing now | Current, not urgent. |
| `PAUSED` | paused | Neutral rest state. |
| `COMPLETED` | loved enough | Warm story-finished state. |
| `FINISHED` | credits rolled | Clear when inferred from story progress. |
| `DROPPED` | released | Permission to let go. |

## Avoid

- "failed"
- "behind"
- "only N left"
- "complete your backlog"
- "streak"
- "overdue"
- "productivity"
- "grind"
- "you haven't played X in N days"
- Copy that treats unplayed games as debt, waste, or a moral problem.

## Prefer

- "Your shelf is quiet right now."
- "This can wait."
- "Pick one for tonight."
- "Still curious."
- "Credits rolled."
- "Ready to release."
- "Try a short session, then decide."

## Examples

| Before | After |
| --- | --- |
| "Stop fighting your backlog." | "Make peace with your shelf." |
| "Wishlist only" | "Stay curious" |
| "Chat failed. Try again." | "Chat paused. Try again when you're ready." |
| "Close enough to finish" | "A short return could be enough" |
| "Your untouched library is already large." | "Your shelf already has plenty of choices." |
| "Move it out of active backlog." | "Release it from the active shelf if it no longer fits." |
